## Why

The Build-a-Bear ritual is the product surface. 7 full-screen stations + lead-settler host + animated transitions + draft persistence + consent gate. Reuses World Builder painted_2d scene for workshop back-room aesthetic. Standard Mode by default; Advanced Mode unlock toggle for power users (advanced UI lives in goal #7). Quick Mode also available (collapses ceremony to 4-screen wizard, same backend).

## Build sequence

### Phase 1 — Route shell + draft persistence
1. Read spec §1, §2 in full.
2. `+page.svelte` + `+page.ts` route handler. Optional `?draftId=...` resumes existing draft.
3. `WorkshopDraftStore.ts`: IDB store `storybook-workshop-drafts-v1`. CRUD per draft. TTL 30 days auto-purge.
4. `KidProfileStore.ts`: parent's kid roster in IDB. CRUD + delete-all-for-kid cascade (also wipes drafts for that kid).
5. `WorkshopOrchestrator.ts`: state machine. States = `kid-picker | s1 | s2 | s3 | s4 | s5 | s6 | s7 | library`. Transitions only allowed forward; back-arrow allowed. Persist after every state change.
6. `stores.ts`: reactive Svelte stores wrapping orchestrator + draft + mode.

### Phase 2 — Workshop scene
7. `WorkshopBackdrop.svelte`: fetches WB painted_2d "back-room print-shop" scene via WB API. Idle animation. Settler attendants positioned per station.
8. `SettlerHostBanner.svelte`: lead settler portrait + speech bubble. Banter per station from existing settler personality infra.
9. `StationProgress.svelte`: 7-dot horizontal progress. Click dots to navigate (back-only).
10. `WalkTransition.svelte`: animated character-walks-to-next-station transition (~600ms).

### Phase 3 — Station 1: Choose Your Story Shape
11. `ThemePicker.svelte`: 12 theme cards (4×3 grid). Each card = small WB-rendered thumbnail + 1-line description.
12. Special occasion chip selector.
13. Length picker: 4 tiers (Bedtime/Standard/Adventure/Saga) with retail-price preview tied to age-band default.
14. `EhriPhaseSelfAssessment.svelte`: 5-question modal. Score → Ehri phase. Optional skip.
15. Carries: `{ theme, occasion, targetSpreads, ehriPhase }` into workshop state.

### Phase 4 — Station 2: Forge Your Hero
16. Photo capture: camera or file picker.
17. Calls `pillarVectorizerService.vectorize(blob)`. Shows progress spinner.
18. Calls `pillarMatcherService.match(vector, { ageBandHint, topK: 8 })`.
19. `PillarMatchCeremony.svelte`: animated spinning wheel ceremony, lands on top-8 grid in default style.
20. Buttons: "More like this", "Different vibe" → refine.
21. Fallback path: WASM init fails → modal w/ consent → POST to `/api/storybook-workshop/vectorize`. Skip option → manual archetype grid.
22. Carries: `pillarId`.

### Phase 5 — Station 3: Wish Moment
23. `DedicationCapture.svelte`: three tabs — Type / Record / Template.
24. Voice record (≤30s) → local blob. Transcribed via Web Speech API.
25. Routes everything through `kidsContentSafetyService.scan` + `privacyFilterService.scrub` before storage.
26. Carries: `{ dedicationText, voiceClipBlob, templateId }`.

### Phase 6 — Station 4: Name Cast
27. `CastBuilder.svelte`: hero name (default from profile), sidekick dropdown (from `agentRegistryService.getAll()` + 3 theme-matched suggestions), pets/sibs add-on, locale text → matched to WB biome.
28. Carries: `{ heroName, sidekickSettlerId, supportingCast, localeBiome }`.

### Phase 7 — Station 5: Dress Your Story
29. `ArtStyleGrid.svelte`: 10-12 style cards. Each renders matched pillar in that style (`GET /api/world/pillar-library/{pillarId}/preview?style=...`).
30. Optional extras toggles: author byline, endpaper, cover badge, easier-reading mode, dialogic prompts (default ON).
31. Carries: `{ artStyle, authorByline?, endpaper?, coverBadge?, easierReadingMode, dialogicPromptsEnabled }`.

### Phase 8 — Station 6: The Seal
32. On entry, full draft state assembled into `StoryInput` → `storyAuthorService.author(...)` → SceneTree.
33. For each scene → `POST /api/world/scene` → spread PNGs.
34. For each spread → `BookSpreadSurfaceAdapter.composite(...)` → composite.
35. `bookAssembler.assemble(bundle)` → `AssembledBook`.
36. `SealAnimation.svelte`: wax-seal ceremony animation while generation in flight (parallelized).
37. `BookFlipPreview.svelte`: page-turn preview using BookSpreadCanvas component (goal #4).
38. Redo affordances surface: per-scene redo, style swap, pillar swap, story reroll, page-budget slider, beat compress/expand.
39. `ConsentGate.svelte`: mandatory dual-checkbox before advancing to Station 7. Captures consent + PDF hash + timestamp into draft.

### Phase 9 — Station 7: Take It Home
40. Two CTAs: "Get free digital" (instant), "Order printed copy" (handed off to goal #8 fulfillment).
41. Format picker + Lulu live shipping quote (via goal #8 backend endpoint).
42. After free-digital: show shareable read-along URL + Share with grandparents CTA + Subscription series gift CTA.
43. Settler delivers banter line + "Make another book?" CTA → loops back to KidPicker.

### Phase 10 — Library
44. `library/+page.svelte`: parent's books per kid grid. Per-book status, tracking link, delete button. Per-kid delete-all (right-to-delete per COPPA-K).
45. Resume in-progress drafts surfaces here.

### Phase 11 — Tests
46. `workshop-draft-store.test.ts`: IDB CRUD, TTL purge, kid-cascade-delete.
47. `workshop-orchestrator.test.ts`: state-machine transitions, can't skip forward, back-only navigation.
48. `station-flow.test.ts`: each station fully completable end-to-end with mocked services.
49. Playwright `e2e/storybook-workshop-flow.spec.ts`: 7-station happy path, draft resume, mode toggle, consent gate enforcement (cannot reach Station 7 without consent checkboxes).

### Phase 12 — Verification
50. `pnpm check` clean.
51. Lint invariants clean (no raw setIntervals, cooperative-yield).
52. `pnpm playwright test e2e/storybook-workshop-flow.spec.ts` → green.
53. Manual smoke in real browser: spawn `pnpm dev:agent` → navigate to `/dashboard/storybook-workshop` → walk through 7 stations with screenshots at each → verify draft resumes after refresh.
